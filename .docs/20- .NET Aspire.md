# 第二十阶段：.NET Aspire

欢迎回到第二十阶段！在本阶段，您将了解 .NET Aspire 以及它如何转变 AI.Boilerplate 项目的开发和部署体验。

## 什么是 .NET Aspire？

.NET Aspire 是一个**有主见的、云就绪的堆栈**，用于构建可观测、生产就绪的分布式应用程序。它是一个以开发人员为先、永远免费、开源的平台，帮助您轻松编排复杂的多服务应用程序。

### 核心理念

Aspire 遵循 **“以代码为中心的控制”** 方法，您可以在类型安全且可读的代码中定义整个堆栈。您可以本地运行并部署到任何地方，而无需更改架构。

---

## .NET Aspire 如何使本项目受益

.NET Aspire **显著改善了开发和部署体验**：

### 1. **开发体验** 🚀

.NET Aspire 提供了一个**出色的仪表板**，用于在开发过程中监控您的应用程序，为您提供：

- **实时日志**：在一个地方查看所有应用程序日志
- **指标**：监控性能计数器和自定义指标
- **分布式追踪**：跨服务跟踪请求
- **健康检查**：查看所有服务的健康状态
- **资源管理**：访问数据库管理工具

此外，它还提供：
- **服务编排**：自动配置和设置依赖项，如数据库、消息代理和存储
- **服务发现**：服务可以相互查找和通信，无需硬编码地址
- **内置开发工具**：
  - **DbGate**：用于数据库管理（在本项目中）
  - **MailPit**：用于电子邮件测试
  - **Minio**：用于兼容 S3 的存储测试
  - **开发隧道 (Dev Tunnels)**：用于公共 HTTPS URL

### 2. **部署灵活性** 🌐

.NET Aspire **简化了部署**，不仅限于 Azure 云，还包括：

- ✅ **Docker Compose**（用于 VPS 和自托管环境的基于容器的部署）
- ✅ **Kubernetes (k8s)**（用于任何提供商的企业级编排）
- ✅ **Azure 云**（自动资源配置）

您为本地开发编写的相同代码适用于所有这些部署目标！

## 本项目中的 Aspire

### 项目结构

Aspire 编排由以下项目管理：

**位置**: [`src/Server/AI.Boilerplate.Server.AppHost/`](/src/Server/AI.Boilerplate.Server.AppHost/)

该项目包含：
- **`Program.cs`**：定义所有应用程序资源和依赖项
- **`appsettings.json`**：通用配置
- **`appsettings.Development.json`**：特定于开发的设置（凭据、密码）

### AppHost 配置

让我们探索 `Program.cs` 文件：

```csharp
using Microsoft.Extensions.Hosting;

var builder = DistributedApplication.CreateBuilder(args);

// 查看 appsettings.Development.json 获取凭据/密码设置。

var sqlDatabase = builder.AddSqlServer("sqlserver")
        .WithDbGate(config => config.WithDataVolume())  // 添加基于 Web 的数据库管理
        .WithDataVolume()  // 在重启间持久化数据
        .WithImage("mssql/server", "2025-latest")  // 带有向量搜索功能的 SQL Server 2025
        .AddDatabase("mssqldb");

// 使用 Minio 进行 S3 兼容存储
var s3Storage = builder.AddMinioContainer("s3")
    .WithDataVolume();  // 持久化上传的文件

// 主服务器项目
var serverWebProject = builder.AddProject("serverweb", "../AI.Boilerplate.Server.Web/AI.Boilerplate.Server.Web.csproj")
    .WithExternalHttpEndpoints();

// 健康检查（仅开发环境出于安全考虑）
if (builder.Environment.IsDevelopment())
{
    serverWebProject.WithHttpHealthCheck("/alive");
}

// 设置依赖项
serverWebProject.WithReference(sqlDatabase).WaitFor(sqlDatabase);
serverWebProject.WithReference(s3Storage);

// 测试项目（仅在运行模式下）
if (builder.ExecutionContext.IsRunMode)
{
    // Blazor WebAssembly 独立版
    builder.AddProject("clientwebwasm", "../../Client/AI.Boilerplate.Client.Web/AI.Boilerplate.Client.Web.csproj")
        .WithExplicitStart();

    // 使用 Mailpit 进行电子邮件测试（仅用于测试目的）
    var mailpit = builder.AddMailPit("smtp")
        .WithDataVolume("mailpit");
    serverWebProject.WithReference(mailpit);

    // Blazor Hybrid Windows
    builder.AddProject("clientwindows", "../../Client/AI.Boilerplate.Client.Windows/AI.Boilerplate.Client.Windows.csproj")
        .WithExplicitStart();

    // 开发隧道用于公共 HTTPS 访问
    var tunnel = builder.AddDevTunnel("web-dev-tunnel")
        .WithAnonymousAccess()
        .WithReference(serverWebProject.WithHttpEndpoint(name: "devTunnel").GetEndpoint("devTunnel"));
}

await builder.Build().RunAsync();
```

### 此配置中的关键特性

1. **SQL Server 2025**：使用最新的 SQL Server，具有嵌入式向量搜索功能
2. **DbGate 集成**：开发期间可访问的基于 Web 的数据库管理 UI
3. **数据卷**：所有数据在容器重启后持久化
4. **服务依赖**：`serverWebProject.WaitFor(sqlDatabase)` 确保数据库准备好后服务器才启动
5. **条件服务**：电子邮件测试和客户端项目仅在开发模式下运行
6. **开发隧道**：允许通过 HTTPS 从外部访问您的本地开发服务器

## Aspire 仪表板

### 什么是 Aspire 仪表板？

Aspire 仪表板是一个**内置的、由 OpenTelemetry 驱动的监控界面**，提供：

- 📊 **实时日志**：在一个地方查看所有应用程序日志
- 📈 **指标**：监控性能计数器和自定义指标
- 🔍 **分布式追踪**：跨服务跟踪请求
- ❤️ **健康检查**：查看所有服务的健康状态
- 🗄️ **资源管理**：访问数据库管理工具

### 如何访问仪表板

当您运行 AppHost 项目时，仪表板会自动在浏览器中打开。您将看到：

1. **资源选项卡**：所有服务、数据库和容器
2. **控制台日志选项卡**：来自所有服务的组合输出
3. **结构化日志选项卡**：可搜索、可过滤的带上下文日志
4. **追踪选项卡**：跨服务边界的分布式追踪
5. **指标选项卡**：性能指标和自定义计数器

### 仪表板优势

✅ **集中监控**：所有应用程序组件的单一界面  
✅ **即时调试**：通过详细日志和追踪快速识别问题  
✅ **无需配置**：开箱即用  
✅ **生产就绪模式**：与生产中使用的可观测性模式相同

## 重要的数据库管理提示

### 挑战

有一个重要的细节可能需要时间才能发现：

当应用程序运行时，您可以通过 **Aspire 内置的 DbGate** 基于 Web 的数据库管理工具管理数据库。

**但是**，当应用程序停止时：
- ❌ 数据库停止
- ❌ DbGate 停止工作
- ❌ 您甚至无法使用外部工具连接（SSMS, Azure Data Studio 等）

这对于测试项目没问题，但对于**真实项目**（您需要持续处理数据库），这可能会受到限制。

### 解决方案：持久化数据库配置

要使数据库**持续运行**（即使应用程序停止）：

1. 打开 `AI.Boilerplate.Server.AppHost` 项目中的 `Program.cs`
2. 向您的数据库设置添加以下配置：

```csharp
var sqlDatabase = builder.AddSqlServer("sqlserver", port: ?????)
    .WithDbGate(config => config.WithDataVolume())
    .WithDataVolume()
    .WithImage("mssql/server", "2025-latest")
    // 🔑 添加这两行以实现持久化：
    .WithLifetime(ContainerLifetime.Persistent)  // ✅ 应用程序停止时保持运行
    .WithEndpointProxySupport(proxyEnabled: false)  // ✅ 直接端口访问
    .AddDatabase("mssqldb");
```

### 结果

✅ 即使应用程序停止，数据库仍保持运行  
✅ 可使用任何数据库工具连接（SSMS, Azure Data Studio, DBeaver 等）  
✅ 使用指定的端口（例如 `localhost,1433`）  
✅ 凭据存储在 `appsettings.Development.json` 中

### 查找数据库凭据

**位置**: [`src/Server/AI.Boilerplate.Server.AppHost/appsettings.Development.json`](/src/Server/AI.Boilerplate.Server.AppHost/appsettings.Development.json)

```json
{
    "Parameters": {
        "sqlserver-password": "P@ssw0rd",
        "sqlserver__Comment": "默认用户名为 `sa`",
        "s3-rootPassword": "P@ssw0rd",
        "s3__Comment": "默认用户名为 `minioadmin`",
        "Comment": "您可能需要删除 docker 卷以应用密码更改"
    }
}
```

**默认 SQL Server 凭据**:
- **用户名**: `sa`
- **密码**: `P@ssw0rd` (可配置)
- **服务器**: `localhost,1433` (或您配置的端口)

## 重要提示：Docker 执行

### 本项目如何使用 Docker

**为了性能优化**:
- ❌ `AI.Boilerplate.Server.Api` **不**在 Docker 上运行
- ❌ `AI.Boilerplate.Server.Web` **不**在 Docker 上运行
- ✅ 它们**原生运行**以获得更快的开发体验

**为什么？**
- 原生执行比容器快得多
- 热重载和调试效果更好
- 减少开发机器上的资源消耗

### 在 Windows 上测试 Linux 行为

如果您想在 **Windows 上使用 Aspire** 的同时 **测试 Linux 中的服务器行为**：

1. 在 **Visual Studio Code** 中打开项目
2. 使用 **Dev Containers** 功能（需要 Docker Desktop 和 Dev Containers 扩展）
3. 在终端中运行此命令以清理之前的 Windows 构建产物：
   ```bash
   chmod +x ./Clean.sh && ./Clean.sh
   ```
4. 然后运行 `AI.Boilerplate.Server.AppHost` 项目

**结果**：您可以在享受 Aspire 功能的同时测试 Linux 行为！🐧

## 使用 Aspire 进行部署

### 支持的部署目标

.NET Aspire 简化了以下部署：

1. **Azure 云**
   - 自动资源配置
   - Azure Container Apps
   - Azure SQL Database
   - Azure Blob Storage

2. **Docker Compose**
   - 自动生成 `docker-compose.yml`
   - 非常适合 VPS 和自托管环境

3. **Kubernetes (k8s)**
   - 生成 Kubernetes 清单
   - 适用于任何 k8s 提供商（Azure AKS, AWS EKS, Google GKE, 本地部署）

### 部署工作原理

Aspire 可以根据您的 `Program.cs` 配置为这些平台生成部署产物。编排本地开发的相同代码可以转换为：

- Azure Bicep 模板
- Docker Compose 文件
- Kubernetes YAML 清单

这确保了开发和生产环境之间的**一致性**！

## ☁️ 走向生产：切换到 Azure 托管服务

虽然本地容器 (Docker) 非常适合开发，但在生产环境中，您可以切换到完全托管的 Azure 服务，以确保可扩展性、安全性和高可用性。

.NET Aspire 通过在 `AppHost/Program.cs` 中交换托管资源而不更改应用程序代码，使这种过渡变得无缝。

### 1. 先决条件包

确保您的 **AppHost** 项目中安装了 Azure 集成包：

```xml
<PackageReference Include="Aspire.Hosting.Azure.Sql" />
<PackageReference Include="Aspire.Hosting.Azure.Redis" />
<PackageReference Include="Aspire.Hosting.Azure.PostgreSQL" />
```

### 2. 更新 AppHost (`Program.cs`)

将本地容器定义替换为 Azure 资源。

#### 🛢️ Azure SQL Database

不使用 `AddSqlServer` (容器)，而使用 `AddAzureSqlServer`：

```csharp
// ❌ 本地容器
// var sqlServer = builder.AddSqlServer("sqlserver");

// ✅ Azure 托管服务
var sqlServer = builder.AddAzureSqlServer("sqlserver")
    .AddDatabase("sqldb");
```

#### ⚡ Azure Cache for Redis

不使用 `AddRedis` (容器)，而使用 `AddAzureRedis`：

```csharp
// ❌ 本地容器
// var redis = builder.AddRedis("redis");

// ✅ Azure 托管服务
var redis = builder.AddAzureRedis("redis");
```

#### 🐘 Azure Database for PostgreSQL (Flexible Server)

要在生产中利用高性能向量搜索，请使用 Azure PostgreSQL Flexible Server。

```csharp
// ❌ 本地容器
// var postgres = builder.AddPostgres("postgres");

// ✅ Azure 托管服务
var postgres = builder.AddAzurePostgresFlexibleServer("postgres")
    .AddDatabase("pgdb");
```
