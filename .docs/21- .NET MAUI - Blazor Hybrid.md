## 什么是 .NET MAUI Blazor Hybrid？

**.NET MAUI (Multi-platform App UI)** 是微软用于构建原生跨平台应用程序的框架。**Blazor Hybrid** 结合了 Blazor Web UI 的强大功能与原生平台能力，使您能够：

- **跨 Web、移动和桌面平台重用 Blazor 组件**
- **访问原生设备功能**，如局域网 TCP 套接字、原生推送通知、更好的文件访问等
- **直接通过 C# 访问原生库**，如 Java、Kotlin、Swift 和 Objective-C 库
- **一次构建，处处运行**：Android、iOS、macOS 和 Windows
- **与 Web 应用程序共享绝大多数代码库**

`AI.Boilerplate.Client.Maui` 和 `AI.Boilerplate.Client.Windows` 项目展示了这一架构的实际应用。

### 平台支持
有关支持的浏览器、操作系统和平台版本的详细信息，请参阅：https://bitplatform.dev/templates

**重要提示**：虽然代码兼容性极佳，但请注意本指南中涵盖的平台特定注意事项。

## 深度链接 (Android) 和通用链接 (iOS)

原生应用最强大的功能之一是能够直接在应用中处理网页链接。

### 什么是深度链接/通用链接？

**目的**：当用户点击 Web 应用链接但其设备上已安装 Android 或 iOS 应用时，应用会直接在特定页面打开，而不是在浏览器中打开。

**用户体验**：假设如果某人安装了您的应用，他们更喜欢使用原生应用而不是 Web 版本。

### 配置要求

#### Android - 深度链接
对于 Android，每个新页面路由都必须添加到 Android 项目的 `MainActivity.cs` 中。

**位置**: [`/src/Client/AI.Boilerplate.Client.Maui/Platforms/Android/MainActivity.cs`](/src/Client/AI.Boilerplate.Client.Maui/Platforms/Android/MainActivity.cs)

**配置示例**:
```csharp
[IntentFilter([Intent.ActionView],
    DataSchemes = ["https", "http"],
    DataHosts = ["use-your-web-app-url-here.com"],
    DataPaths = [PageUrls.Home],
    DataPathPrefixes = [
        "/en-US", "/en-GB", /* ... 其他语言环境 */,
        PageUrls.SignIn, PageUrls.SignUp, /* ... 其他页面路由 */
    ],
    AutoVerify = true,
    Categories = [Intent.ActionView, Intent.CategoryDefault, Intent.CategoryBrowsable])]
```

**重要提示**：每次向应用添加新页面时，必须将其路由添加到 `DataPathPrefixes` 数组中。

#### iOS - 通用链接
对于 iOS，这是由 `AI.Boilerplate.Client.Web` 项目的 `wwwroot` 中的 `apple-app-site-association` 文件自动处理的。**无需**为每个页面进行手动配置！

**位置**: [`/src/Client/AI.Boilerplate.Client.Web/wwwroot/.well-known/apple-app-site-association`](/src/Client/AI.Boilerplate.Client.Web/wwwroot/.well-known/apple-app-site-association)

**工作原理**：iOS 会自动从您的网站读取此文件并处理通用链接，无需逐页配置。

### 重要的设置步骤

在深度/通用链接生效之前，您必须：

1. **更新 iOS 的 `apple-app-site-association` 文件中的 `appId`**
2. **更新 Android 的 `assetlinks.json` 文件**
3. **在 Google Play 商店和 Apple App Store 上发布您的应用**

**注意**：深度链接和通用链接仅在应用发布到相应商店后才有效。

## 原生应用的应用程序版本控制

### AI.Boilerplate.Client.Maui 中的 ApplicationVersion

`AI.Boilerplate.Client.Maui.csproj` 文件中的 `ApplicationVersion` 属性管理应用商店提交和更新的应用版本。

**目的**：Google Play 和 Apple Store 要求的整数版本号（无小数点或点）。

**自动生成**：项目会根据 `Version` 属性自动生成 `ApplicationVersion`：
- **示例**：如果 `Version` 是 `1.7.3`，则 `ApplicationVersion` 变为 `10703`

```xml
<!-- 在 AI.Boilerplate.Client.Maui.csproj 中 -->
<Version>1.7.3</Version>
<!-- ApplicationVersion 自动计算为 10703 -->
```

这种整数格式是移动平台进行正确版本比较所必需的。

## AI.Boilerplate.Client.Windows 的优势

`AI.Boilerplate.Client.Windows` 项目相比 MAUI 的 Windows 输出具有若干优势：

### 主要优势

1. **更广泛的操作系统支持**：适用于 Windows 7、8、10 和 11
   - 相比之下，`AI.Boilerplate.Client.Maui` 的 Windows 输出仅适用于 Windows 10 和 11

2. **更好的性能**：比 MAUI Windows 版本执行更快

3. **自动更新**：内置自动更新功能

### 何时使用哪一个？
- **使用 Client.Windows**：为了获得最大的 Windows 兼容性和最佳性能
- **使用 Client.Maui**：当您需要在包括 Windows 在内的所有平台上拥有统一的代码库时

## WebView 版本的重要性

理解 WebView 对于 Blazor Hybrid 应用至关重要。

### 关键概念

**关键组件**：除了操作系统版本（Windows、Android、iOS、macOS）外，**WebView 版本**也至关重要。

**为什么重要**：UI 在 WebView 内使用 HTML/CSS 运行：
- 您的 Blazor 组件在此 WebView 内渲染为 HTML
- 现代 Web 功能需要最新的 WebView 版本
- 过时的 WebView 可能导致兼容性问题

**完全原生访问**：尽管 UI 在 WebView 中渲染，但 MAUI/Blazor Hybrid 中的 C# .NET 拥有**完全访问**原生操作系统功能的权限：
- 您可以直接在 C# 中使用 Java/Swift/Objective-C/Kotlin 代码
- 您可以从 Maven（类似于 NuGet）安装包并直接在 C# .NET 中使用它们
- 原生 API 访问没有限制

**用户指导**：用户应保持其 WebView 更新以获得最佳兼容性和安全性。

### WebView 版本管理

应用在 [`App.xaml.cs`](/src/Client/AI.Boilerplate.Client.Maui/App.xaml.cs) 中检查 Android 上的 WebView 版本以确保兼容性：

```csharp
// 在 App.xaml.cs 的 OnStart() 中
#if Android
const int minimumSupportedWebViewVersion = 85;

if (Version.TryParse(Android.Webkit.WebView.CurrentWebViewPackage?.VersionName, 
    out var webViewVersion) 
    && webViewVersion.Major < minimumSupportedWebViewVersion)
{
    var webViewName = Android.Webkit.WebView.CurrentWebViewPackage.PackageName;
    
    await App.Current!.Windows[0].Page!.DisplayAlertAsync(
        "AI.Boilerplate", 
        localizer[nameof(AppStrings.UpdateWebViewThroughGooglePlay)], 
        localizer[nameof(AppStrings.Ok)]);
    
    await Launcher.OpenAsync(
        $"https://play.google.com/store/apps/details?id={webViewName}");
}
#endif
```

**为什么重要**：较旧的 Android 设备可能拥有不支持现代 Web 功能的过时 WebView 版本。此检查确保良好的用户体验。

## 理解架构

### 1. 入口点：MauiProgram.cs

[`MauiProgram.cs`](/src/Client/AI.Boilerplate.Client.Maui/MauiProgram.cs) 文件是 MAUI 应用程序的起点：

```csharp
public static MauiApp CreateMauiApp()
{
    var builder = MauiApp.CreateBuilder();
    
    builder
        .UseMauiApp<App>()
        .UseAppStoreInfo()
        .UseSentry(options => { /* ... */ })
        .Configuration.AddClientConfigurations(clientEntryAssemblyName: "AI.Boilerplate.Client.Maui");
    
    builder.ConfigureServices();
    
    builder.ConfigureLifecycleEvents(lifecycle =>
    {
        // 平台特定的生命周期事件
    });
    
    SetupBlazorWebView();
    
    return builder.Build();
}
```

**关键特性**：
- **异常处理**：用于 `AppDomain.UnhandledException` 和 `TaskScheduler.UnobservedTaskException` 的全局异常处理程序
- **平台检测**：设置 `AppPlatform.IsBlazorHybrid = true`
- **配置**：从 `appsettings.json` 文件加载配置
- **服务注册**：调用 `ConfigureServices()` 注册所有依赖项
- **深度链接**：配置通用链接处理（iOS/macOS）
- **BlazorWebView 设置**：为每个平台自定义 WebView

### 2. 服务注册：MauiProgram.Services.cs

MAUI 特定的服务在 [`MauiProgram.Services.cs`](/src/Client/AI.Boilerplate.Client.Maui/MauiProgram.Services.cs) 中注册：

```csharp
public static void ConfigureServices(this MauiAppBuilder builder)
{
    var services = builder.Services;
    var configuration = builder.Configuration;
    
    // 共享的 client-core 服务（从 Web 重用）
    services.AddClientCoreProjectServices(configuration);
    
    // MAUI 特定的服务
    services.AddScoped<IWebAuthnService, MauiWebAuthnService>();
    services.AddScoped<IExceptionHandler, MauiExceptionHandler>();
    services.AddScoped<IAppUpdateService, MauiAppUpdateService>();
    services.AddScoped<IBitDeviceCoordinator, MauiDeviceCoordinator>();
    services.AddScoped<IExternalNavigationService, MauiExternalNavigationService>();
    
    // 用于持久数据的存储服务
    services.AddSingleton<IStorageService, MauiStorageService>();
    
    // 配置正确的 HTTP 客户端
    services.AddScoped<HttpClient>(sp => { /* ... */ });
    
    // 平台特定的服务（Android, iOS, Windows, macOS）
#if Android
    services.AddClientMauiProjectAndroidServices(builder.Configuration);
#elif iOS
    services.AddClientMauiProjectIosServices(builder.Configuration);
    // ... 等等
#endif
}
```

**重要提示**：请注意项目如何重用 `AddClientCoreProjectServices()` —— 这是 Web 和 MAUI 应用共用的**共享 Blazor 组件库**！

### 3. BlazorWebView：原生与 Web 之间的桥梁

[`MainPage.xaml`](/src/Client/AI.Boilerplate.Client.Maui/MainPage.xaml) 文件托管 `BlazorWebView` 组件，该组件在原生 WebView 内渲染 Blazor 内容：

```xml
<ContentPage xmlns:b="clr-namespace:Microsoft.AspNetCore.Components.WebView.Maui;...">
    <b:BlazorWebView
        x:Name="AppWebView"
        HostPage="wwwroot/index.html">
        <b:BlazorWebView.RootComponents>
            <b:RootComponent 
                ComponentType="{x:Type core:Routes}" 
                Selector="#app-container" />
        </b:BlazorWebView.RootComponents>
    </b:BlazorWebView>
</ContentPage>
```

**工作原理**：
- `BlazorWebView` 组件为每个平台创建一个原生 WebView
- 它加载 `wwwroot/index.html` 文件
- 它在 `#app-container` 元素内渲染 `Routes` 组件（来自 `AI.Boilerplate.Client.Core`）
- 您所有的 Blazor 组件都在此 WebView 内渲染，就像在浏览器中一样！

### 4. BlazorWebView 自定义

[`MauiProgram.cs`](/src/Client/AI.Boilerplate.Client.Maui/MauiProgram.cs) 中的 `SetupBlazorWebView()` 方法为每个平台自定义 WebView：

```csharp
private static void SetupBlazorWebView()
{
    BlazorWebViewHandler.BlazorWebViewMapper.AppendToMapping("CustomBlazorWebViewMapper", 
        static (handler, view) =>
    {
        var webView = handler.PlatformView;
        
#if Windows
        // Windows 特定的 WebView2 配置
        webView.DefaultBackgroundColor = /* ... */;
        webView.CoreWebView2.PermissionRequested += /* ... */;
        
#elif iOS || Mac
        // iOS/macOS WKWebView 配置
        webView.Configuration.AllowsInlineMediaPlayback = true;
        webView.ScrollView.Bounces = false;
        webView.SetValueForKey(NSObject.FromObject(true), new NSString("inspectable"));
        
#elif Android
        // Android WebView 配置
        webView.Settings.AllowFileAccess = true;
        webView.Settings.DomStorageEnabled = true;
#endif
    });
}
```

**平台特定功能**：
- **Windows**：启用权限，在生产环境中禁用缩放
- **iOS/macOS**：启用 Web 检查器，允许内联媒体播放
- **Android**：在开发环境中启用文件访问、DOM 存储和混合内容

## 平台特定服务

MAUI 项目包含共享接口的平台特定实现：

### MauiStorageService

在 [`MauiStorageService.cs`](/src/Client/AI.Boilerplate.Client.Maui/Infrastructure/Services/MauiStorageService.cs) 中使用 MAUI 的 `Preferences` API 实现 `IStorageService`：

```csharp
public partial class MauiStorageService : IStorageService
{
    public async ValueTask<string?> GetItem(string key)
    {
        return Preferences.Get(key, null);
    }
    
    public async ValueTask SetItem(string key, string? value, bool persistent = true)
    {
        if (persistent)
        {
            Preferences.Set(key, value); // 持久化到磁盘
        }
        else
        {
            tempStorage[key] = value; // 仅在内存中
        }
    }
}
```

**用途**：存储用户偏好设置、身份验证令牌以及其他应在应用重启后持久存在的数据。

### MauiExternalNavigationService

在 [`MauiExternalNavigationService.cs`](/src/Client/AI.Boilerplate.Client.Maui/Infrastructure/Services/MauiExternalNavigationService.cs) 中实现 `IExternalNavigationService` 以打开外部 URL：

```csharp
public partial class MauiExternalNavigationService : IExternalNavigationService
{
    public async Task NavigateToAsync(string url)
    {
        await Browser.OpenAsync(url, options: new()
        {
            TitleMode = BrowserTitleMode.Hide,
            PreferredToolbarColor = /* ... */,
            // Windows/macOS: 在系统浏览器中打开
            // Android/iOS: 在应用内浏览器中打开
            LaunchMode = AppPlatform.IsWindows || AppPlatform.IsMacOS 
                ? BrowserLaunchMode.External 
                : BrowserLaunchMode.SystemPreferred
        });
    }
}
```

**用途**：在适当的浏览器中打开外部链接（如服务条款、隐私政策）。
